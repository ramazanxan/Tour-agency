# Jolu — туристический маркетплейс Кыргызстана 🏔

> Booking.com для туров по Кыргызстану. Туристы находят, сравнивают и бронируют
> туры в одном месте; туркомпании управляют турами и заявками из кабинета.

Монорепозиторий: **веб-приложение (Next.js)** + **Telegram-бот (aiogram)** + **схема Supabase**.

---

## 📦 Структура

```
Tour-agency/
├── web/                # Next.js 14 (App Router) + Tailwind — сайт
│   └── src/
│       ├── app/        # страницы: главная, каталог, тур, бронирование, кабинет, админка
│       ├── components/ # UI и блоки
│       └── lib/        # типы, мок-данные, supabase-клиент, утилиты
├── bot/                # Telegram-бот (Python 3.12 + aiogram 3.x)
├── supabase/
│   └── schema.sql      # таблицы, RLS, индексы, триггеры
└── README.md
```

## ✅ Что реализовано (Фаза 1 / MVP)

**Туристу:**
- Главная: hero с поиском, категории, «Сезон сейчас», «Горящие туры», направления
- Каталог с фильтрами (тип, направление, цена, длительность, сложность, язык гида, особенности) и сортировкой
- Страница тура: галерея, программа по дням, что включено/нет, чек-лист, карта точки сбора (Leaflet/OSM), карточка компании, похожие туры
- Бронирование без онлайн-оплаты: выбор даты выезда, кол-во человек, контакты → заявка
- Личный кабинет: мои брони со статусами, подключение Telegram

**Туркомпании:**
- Кабинет (мини-CRM): заявки со сменой статуса (новая → подтверждена → предоплата → завершена)
- Список туров, календарь выездов с загрузкой мест и цветовой индикацией
- Телефон туриста открывается только после подтверждения заявки

**Админу:**
- Модерация компаний (верификация) и туров, сводная аналитика

**Бот:** привязка аккаунта по deep link, уведомления о заявках и статусах.

> Чаты, отзывы, геймификация, подписки и онлайн-оплата — **Фаза 2–3** (см. ТЗ).

## 🚀 Запуск веб-приложения

```bash
cd web
npm install
cp .env.example .env.local   # можно оставить пустым — сайт работает на мок-данных
npm run dev                  # http://localhost:3000
```

Сайт **запускается без бэкенда**: пока переменные Supabase не заданы, данные берутся
из `web/src/lib/mock-data.ts`, а заявки принимаются в демо-режиме.

### Подключение Supabase (боевой режим)
1. Создайте проект на [supabase.com](https://supabase.com).
2. Выполните `supabase/schema.sql` в SQL Editor.
3. Заполните в `web/.env.local`:
   ```
   NEXT_PUBLIC_SUPABASE_URL=...
   NEXT_PUBLIC_SUPABASE_ANON_KEY=...
   ```

## 🤖 Запуск Telegram-бота

```bash
cd bot
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env        # впишите BOT_TOKEN от @BotFather
python bot.py
```

Бот и сайт работают с одной БД Supabase. Уведомления отправляются при новой заявке
(через `pg_notify('new_booking')` / Edge Function).

## ☁️ Деплой

- **Web → Vercel:** импортируйте репозиторий, Root Directory = `web`, задайте env-переменные.
- **Bot → Railway/Render:** Root = `bot`, команда `python bot.py`, добавьте env.
