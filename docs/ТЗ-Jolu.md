1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА
1.1 Суть продукта
Jolu (кырг. «жол» — путь, дорога) — онлайн-платформа, объединяющая туристов и туроператоров Кыргызстана. Туристы находят, сравнивают и бронируют туры в одном месте; туркомпании получают кабинет для управления турами, заявками и клиентами.


Одной фразой: Booking.com для туров по Кыргызстану.
1.2 Проблема, которую решаем
Туристы: туры разбросаны по десяткам Instagram-аккаунтов и Telegram-каналов. Нет сравнения цен, нет отзывов, нет гарантий, нет понимания свободных мест. Бронирование = «напишите в директ».


Туркомпании: ведут заявки в 40 переписках WhatsApp/Instagram, теряют клиентов, тратят часы на постинг одинакового контента, не имеют статистики.


Иностранные туристы: ищут «Kyrgyzstan tours» в Google и находят устаревшие сайты. Платёжеспособный сегмент остаётся недообслуженным.

2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ — ВЕБ-ПРИЛОЖЕНИЕ
2.1 Сторона туриста
2.1.1 Главная страница
* Hero-блок: поисковая строка («Куда хотите поехать?») + фоновое видео гор (автоплей, muted)
* Быстрые категории-чипсы: 🏔 Треккинг · 🐎 Конные туры · 🚙 Джип-туры · ⛺ Кемпинг · 🏖 Иссык-Куль · 🌍 За рубеж
* Блок «Сезон сейчас»: подборка актуальных направлений месяца (июнь → Алтын-Арашан, Сказка; июль → Сон-Куль)
* Карусель «Горящие туры» — туры с ближайшим выездом и свободными местами
* Блок «Популярные направления» — карточки локаций с числом туров
* Отзывы недели с фото
* Карта Кыргызстана с пинами направлений (интерактивная)
2.1.2 Каталог туров
* Фильтры (sticky-панель слева / снизу на мобиле):
   * Направление (мультивыбор)
   * Даты (календарь, диапазон)
   * Цена (слайдер, сом/USD)
   * Длительность (1 день / выходные / 3–7 дней / 7+)
   * Тип: треккинг, конный, джип, кемпинг, культурный, гастро, горнолыжный
   * Сложность: лёгкий / средний / сложный (с пояснением)
   * Язык гида: KG / RU / EN
   * Особенности: можно с детьми, всё включено, маленькая группа (до 8 чел)
* Сортировка: популярные / цена ↑↓ / ближайший выезд / рейтинг
* Карточка тура в каталоге: фото, название, локация, длительность, цена «от», рейтинг ⭐, ближайшая дата, бейдж «Осталось 3 места» / «Гарантированный выезд»
* Переключатель «Список / Карта»
2.1.3 Страница тура
* Галерея: фото + видео (вертикальные видео поддерживаются — компании снимают для Reels)
* Ключевая информация (карточка справа, sticky):
   * Цена за человека, что входит в цену
   * Календарь дат выезда с количеством свободных мест
   * Кнопка «Забронировать» + «Задать вопрос»
   * Кнопка «❤ В избранное» и «Поделиться»
* Программа по дням (аккордеон с таймлайном)
* Что включено / не включено (две колонки, ✅ / ❌)
* Что взять с собой (чек-лист)
* Точка сбора на карте + время
* Карточка компании: лого, рейтинг, число проведённых туров, бейдж «Верифицирована ✓», ссылка на профиль
* Отзывы: рейтинг по критериям (организация / гид / транспорт / соответствие описанию), фото туристов
* Похожие туры
* Блок «Сбор группы»: «Выезд состоится при наборе 8 человек. Уже записались: 5. Пригласи друзей — при полной группе цена снизится на 10%» + кнопка шеринга
2.1.4 Бронирование
* Шаг 1: дата, количество человек (взрослые/дети), контакты
* Шаг 2: подтверждение заявки → уведомление компании
* Статусы брони: ожидает подтверждения → подтверждена → предоплата → завершена / отменена
* MVP: оплата напрямую компании (предоплата по реквизитам, платформа фиксирует факт). Фаза 2: онлайн-оплата (Mbank/Элсом/FreedomPay, карта)
* Автоматические напоминания: за 3 дня и за 1 день до тура (email + Telegram)
2.1.5 Личный кабинет туриста
* Мои бронирования (активные / прошедшие) со статусами
* Избранное
* Мои отзывы
* Профиль и настройки уведомлений
* «Паспорт путешественника»: карта КР, где закрашиваются посещённые места + бейджи-достижения («Покоритель Сон-Куля», «5 туров за сезон») — геймификация для удержания и шеринга
2.1.6 Отзывы
* Оставить отзыв можно только после завершённого тура (защита от фейков)
* Оценка по 4 критериям + текст + до 10 фото
* Компания может публично ответить
* Жалоба на отзыв → модерация
2.1.7 Чат турист ↔ компания
* Встроенный чат на странице тура и в кабинете
* Уведомления о новых сообщениях в Telegram
* Быстрые вопросы-шаблоны: «Есть ли места на ближайшие даты?», «Подходит ли тур для детей?»
2.2 Сторона туркомпании
2.2.1 Регистрация и верификация
* Регистрация: название, контакты, описание, лого, документы (свидетельство о регистрации)
* Модерация админом → бейдж «Верифицирована ✓»
* Профиль компании: публичная страница со всеми турами, рейтингом, отзывами
2.2.2 Управление турами
* Конструктор тура (пошаговый, с автосохранением):
   1. Основное: название, тип, направление, сложность, длительность
   2. Программа по дням (редактор)
   3. Цены: за человека, детская цена, групповые скидки, что включено
   4. Медиа: до 20 фото + видео (drag & drop, авто-сжатие)
   5. Даты выездов: календарь, мест всего / занято, минимум для выезда
   6. Детали: точка сбора, что взять, требования
* AI-помощник: компания вводит черновик описания → AI улучшает текст, генерирует EN-версию, предлагает чек-лист «что взять»
* Дублирование тура (для повторяющихся туров — главный сценарий)
* Статусы: черновик → на модерации → опубликован → архив
2.2.3 Управление заявками (мини-CRM)
* Канбан или список: новая → подтверждена → предоплата → завершена → отменена
* Карточка заявки: турист, контакты, тур, дата, число человек, история переписки
* Заметки по клиенту
* Массовые действия: подтвердить, напомнить о предоплате
2.2.4 Календарь выездов
* Сводный календарь всех туров с загрузкой мест
* Цветовая индикация: 🟢 набирается / 🟡 почти полный / 🔴 мест нет / ⚪ под угрозой отмены
2.2.5 Статистика (PRO)
* Просмотры туров, конверсия просмотр → заявка
* Источники переходов
* Динамика бронирований по неделям
* Сравнение своих туров между собой
2.2.6 Автопостинг (PRO, фаза 2)
* Тур опубликован → платформа генерирует готовый пост (фото + текст + хэштеги) для Instagram/Telegram
* Кнопка «Скопировать пост» в MVP; автопубликация через API — позже
2.3 Админ-панель
* Модерация компаний (документы, верификация)
* Модерация туров и отзывов
* Управление направлениями и категориями
* Сводная аналитика платформы: GMV, заявки, конверсии, топ туров
* Управление главной страницей (сезонные подборки, баннеры)
* Логи и поддержка пользователей

3. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ — TELEGRAM-БОТ
Бот — не дубль сайта, а канал уведомлений + быстрых сценариев. Стек: Python + aiogram 3.x.
3.1 Для туристов
* /start → привязка аккаунта (deep link с сайта)
* 🔍 Быстрый поиск: бот спрашивает направление → даты → бюджет → выдаёт 3–5 карточек туров (фото + цена + кнопка «Открыть на сайте» / «Забронировать»)
* 🔔 Уведомления: статус брони изменился, напоминание о туре (за 3 дня / 1 день), ответ компании в чате
* ⚡ Подписка на направление: «Сообщи, когда появится тур в Сон-Куль до 5000 сом» → бот присылает при появлении (киллер-фича удержания)
* 🌡 «Куда поехать на выходных?» — кнопка, выдаёт подборку на ближайшие сб-вс с учётом сезона
* Мои брони: список активных с статусами
3.2 Для туркомпаний
* 🔔 Мгновенные уведомления о новых заявках (с кнопками «Подтвердить» / «Открыть в кабинете»)
* Уведомления о новых сообщениях в чате и новых отзывах
* Ежедневная сводка утром: «Сегодня: 2 выезда, 14 человек. Новых заявок за вчера: 5»
* ⚠️ Алерты: «Тур 14.06 под угрозой: набрано 4 из 8, осталось 5 дней»
3.3 Telegram Mini App (фаза 2)
* Каталог туров внутри Telegram без перехода на сайт
* Особенно важно для аудитории КР/КЗ, где Telegram — основной мессенджер

4. ТРЕБОВАНИЯ К ДИЗАЙНУ И UX
4.1 Общие принципы
* Mobile-first: 80%+ трафика будет с телефонов
* Визуальный язык: природа КР — глубокий синий (озёра), белый (вершины), тёплый оранжевый (закаты/юрты) как акцент
* Большие сочные фотографии — продукт продаётся глазами
* Минимум шагов до заявки: гость может забронировать без регистрации (телефон + имя), аккаунт создаётся автоматически
* Скорость: каталог открывается < 2 сек, изображения lazy-load + WebP
4.2 Что должно «манить»
* Видео-хедеры на страницах направлений
* Микроанимации: сердечко избранного, прогресс набора группы (заполняющаяся шкала «5/8 мест»)
* Социальные триггеры: «12 человек смотрят этот тур», «Забронировано 3 раза за сегодня» (честные, на реальных данных)
* Шеринг-карточки: при «Поделиться» генерируется красивая OG-карточка с фото, ценой и датой
* Тёмная тема (фаза 2)
4.3 Локализация
* MVP: русский
* Фаза 2: кыргызский, английский (EN критичен для иностранного сегмента)
* Цены: сом / USD переключатель

5.3 Интеграция сайт ↔ бот
* Привязка: на сайте кнопка «Подключить Telegram» → deep link t.me/jolu_bot?start=<jwt_token> → бот сохраняет tg_chat_id
* Бот и сайт работают с одной БД Supabase
* Уведомления: триггеры/Edge Functions Supabase → отправка через Bot API
5.4 Безопасность
* Supabase RLS (Row Level Security) на всех таблицах: компания видит только свои заявки
* Rate limiting на формы заявок (защита от спама)
* Телефоны туристов открываются компании только после подтверждения заявки
* Загрузка фото: проверка типа/размера, авто-сжатие

9. ЗАПУСК И ПЕРВЫЕ ПОЛЬЗОВАТЕЛИ
1. Компании (до запуска): лично обойти/обзвонить 20–30 туроператоров Бишкека. Оффер: «Бесплатно размещаем ваши туры, приводим заявки, вы ничего не делаете — мы сами всё загрузим». Цель — 10 «да».
2. Туристы (с запуска):
   * TikTok/Reels: видео туров с озвучкой «куда поехать на выходных за 3000 сом» → ссылка на каталог
   * Telegram-каналы Бишкека (реклама в «куда сходить»-каналах — дёшево)
   * SEO: страницы направлений («Тур в Алтын-Арашан цена», «Сон-Куль тур 2026») — низкая конкуренция, быстрый эффект
3. Сезонный хук: запуск в июне-июле = пик спроса, компании максимально сговорчивы.
